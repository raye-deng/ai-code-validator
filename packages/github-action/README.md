# Open Code Review — GitHub Action

**AI code quality gate for your pull requests** — Detect hallucinations, stale APIs, logic gaps, and context coherence issues automatically.

[![GitHub Marketplace](https://img.shields.io/badge/Marketplace-Open%20Code%20Review-blue?logo=github)](https://github.com/marketplace/actions/open-code-review)
[![License: BSL 1.1](https://img.shields.io/badge/License-BSL%201.1-blue.svg)](https://github.com/raye-deng/open-code-review/blob/main/LICENSE)

## ✨ Features

- 🔍 **3 Scan Levels**:
  - **L1**: Structural analysis (AST-based, fast)
  - **L2**: + Embedding recall (local Ollama or TF-IDF fallback)
  - **L3**: + LLM deep scan (Ollama/OpenAI/Anthropic)
- 🎯 **AI-Specific Defects**:
  - Hallucinated packages/APIs
  - Stale/phantom dependencies
  - Context coherence issues (unused code, broken refs)
  - Logic gaps in AI-generated code
- 📊 **PR Comments**: Automated quality score + issue list
- ⚡ **Diff Mode**: Scan only changed files (fast)
- 🔒 **100% Local**: Optional L2/L3 with Ollama (no cloud required)

## 📦 Usage

### Basic (L1 - Structural Analysis)

```yaml
name: Code Review

on:
  pull_request:
    types: [opened, synchronize, reopened]

permissions:
  contents: read
  pull-requests: write

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Open Code Review
        uses: raye-deng/open-code-review@v1
        with:
          sla: L1
          threshold: 70
          scan-mode: diff
          github-token: ${{ secrets.GITHUB_TOKEN }}
```

### Advanced (L2 with Ollama)

```yaml
- name: Open Code Review (L2)
  uses: raye-deng/open-code-review@v1
  with:
    sla: L2
    threshold: 80
    scan-mode: diff
    ollama-url: ${{ secrets.OLLAMA_URL }}  # Optional: http://your-ollama:11434
    github-token: ${{ secrets.GITHUB_TOKEN }}
```

### Full Scan (L3 with LLM)

```yaml
- name: Open Code Review (L3)
  uses: raye-deng/open-code-review@v1
  with:
    sla: L3
    threshold: 85
    scan-mode: full
    ollama-url: ${{ secrets.OLLAMA_URL }}
    fail-on-low-score: false  # Don't fail on low score
    github-token: ${{ secrets.GITHUB_TOKEN }}
  env:
    OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}  # For L3 if no Ollama
```

## ⚙️ Inputs

| Input | Description | Required | Default |
|-------|-------------|----------|---------|
| `sla` | Scan level: `L1` (structural), `L2` (+ embedding), `L3` (+ LLM) | No | `L1` |
| `threshold` | Minimum score to pass (0-100) | No | `70` |
| `scan-mode` | `diff` (changed files) or `full` | No | `diff` |
| `ollama-url` | Ollama API URL for L2/L3 | No | (uses TF-IDF fallback) |
| `fail-on-low-score` | Fail workflow if score < threshold | No | `true` |
| `github-token` | GitHub token for PR comments | No | `${{ github.token }}` |

## 📤 Outputs

| Output | Description |
|--------|-------------|
| `score` | Quality score (0-100) |
| `grade` | Letter grade (A-F) |
| `issues-count` | Total issues found |

### Access Outputs

```yaml
- name: Open Code Review
  id: review
  uses: raye-deng/open-code-review@v1
  with:
    sla: L1

- name: Show results
  run: |
    echo "Score: ${{ steps.review.outputs.score }}"
    echo "Grade: ${{ steps.review.outputs.grade }}"
    echo "Issues: ${{ steps.review.outputs.issues-count }}"
```

## 🛠️ Requirements

- **Node.js**: 20+ (for GitHub Action runner)
- **For L2/L3** (optional):
  - Ollama server with `nomic-embed-text` (L2) and `qwen3-coder` (L3)
  - OR OpenAI/Anthropic API key

## 📊 Scan Levels Comparison

| Level | Speed | Accuracy | AI Required | Best For |
|-------|-------|----------|-------------|----------|
| L1 | ⚡ Fast (~5s) | Good | No | Quick PR checks |
| L2 | 🚀 Medium (~30s) | Better | Optional (Ollama) | Team projects |
| L3 | 🐢 Slow (~2min) | Best | Yes (Ollama/Cloud) | Critical code |

## 🔒 Privacy & Security

- **L1**: 100% local, no external API calls
- **L2/L3**: Optional Ollama (self-hosted) or cloud APIs
- **No code storage**: Scans run in CI, results not persisted
- **Open source**: Full source available on GitHub

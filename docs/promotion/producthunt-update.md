## Product Name
Open Code Review

## Tagline
The first CI/CD quality gate built for AI-generated code. Free, open-source, runs 100% locally.

## Description (Short)
AI coding assistants generate code with defects that traditional tools miss — hallucinated imports, deprecated APIs, security anti-patterns. Open Code Review catches them all in under 10 seconds, for free, without sending your code anywhere.

## Description (Full)

### The Problem

Teams using AI coding assistants (Copilot, Cursor, Claude Code) ship faster but introduce a new category of bugs:

- **Hallucinated imports** — AI references packages that don't exist on npm/PyPI
- **Deprecated APIs** — Code uses APIs from training data that are now broken
- **Security anti-patterns** — Hardcoded secrets, eval(), SQL injection
- **Over-engineering** — Unnecessary abstractions generated by AI
- **Context artifacts** — Logic contradictions across AI-generated files

Traditional linters check syntax, not reality. They can't tell if a package exists on npm.

### The Solution

Open Code Review is a free, open-source CLI tool with a two-level scanning engine:

**L1 — Pattern Detection (no AI needed, ~3 seconds)**
- Registry verification for every import (npm/PyPI)
- Deprecated API detection (AST-based)
- Security anti-pattern matching
- Code duplication analysis

**L2 — AI Deep Analysis (local Ollama, zero API cost)**
- Embedding-based semantic analysis
- LLM-powered code quality review
- Cross-file coherence checking
- Enhanced scoring with AI confidence

### Key Differentiators

- 🏠 **100% local** — code never leaves your machine
- 💰 **Zero cost** — no API keys needed for L1; Ollama for free L2
- 🔗 **CI/CD native** — GitHub Action + GitLab CI + SARIF output
- 📦 **Registry-aware** — checks imports against real package registries
- 🛡️ **Phantom Package DB** — tracks commonly hallucinated package names

### Quick Start

```bash
npx @opencodereview/cli scan . --sla L1
```

### GitHub Action

```yaml
- uses: raye-deng/open-code-review@v1
  with:
    sla: L1
    threshold: 60
```

## Launch Comment

We've been using AI assistants heavily and kept running into the same bugs — hallucinated imports to packages that don't exist. Traditional tools can't catch these because they don't query the registry.

Open Code Review fills this gap. L1 is instant and free (no AI needed). L2 adds deep analysis with local Ollama (still free). Everything runs on your machine — no data leaves.

We're actively building the Phantom Package DB to track commonly hallucinated names. If you've ever seen `import x from 'some-package'` where some-package doesn't exist, we'd love to know about it.

Try it: `npx @opencodereview/cli scan . --sla L1`
